📚 Linked List Basics in C

Understand pointers, structs, heap memory, and the foundation of linked lists in C.

🔗 What is a Linked List?

A linked list is a linear data structure where each element (called a node) contains data and a pointer to the next node. Unlike arrays, nodes are not stored contiguously in memory.

In C, a linked list is implemented using structs and pointers. The last node's next pointer is NULL.

🗺️ Visual Structure
Normal node
NULL terminator
✅ Advantages

• Dynamic size — grows/shrinks at runtime

• O(1) insertion/deletion at head

• No memory waste (only allocate what you need)

• Foundation for stacks, queues, graphs

❌ Disadvantages

• No random access — O(n) to reach position k

• Extra memory for pointer per node

• Poor cache locality vs arrays

• Risk of memory leaks if free() not called

🔭 What is a Pointer?

A pointer is a variable that stores the memory address of another variable. In C, pointers are central to linked lists because each node must "point to" the next.

int x = 42; // regular int variable at address 0x200 int* ptr = &x; // ptr stores address of x → 0x200 printf("Value: %d\n", *ptr); // dereference → 42 printf("Address: %p\n", ptr); // → 0x200 *ptr = 100; // changes x to 100 via pointer
🧠 Pointer Visualization
📌 Key Operators
int x = 5; int* p = &x; // & = address-of int v = *p; // * = dereference // For struct pointers: p->data // same as (*p).data
⚠️ NULL Pointer
int* p = NULL; // safe init if (p != NULL) { printf("%d", *p); // safe } // NEVER do this: *p = 5; // → SEGFAULT! NULL deref
⚠️
Golden Rule: Always check if a pointer is NULL before dereferencing it. Dereferencing NULL is the #1 cause of segmentation faults in C linked list code.
🏗️ Node Definition in C

A linked list node is defined as a struct in C. It contains the data and a self-referential pointer to the next node.

#include <stdio.h> #include <stdlib.h> /* Define the node structure */ struct Node { int data; // 4 bytes — actual value struct Node* next; // 8 bytes (64-bit) — address of next node }; /* Typedef for convenience */ typedef struct Node Node; /* Create a new node */ Node* createNode(int val) { Node* n = (Node*) malloc(sizeof(Node)); if (n == NULL) { /* malloc failed — always check! */ fprintf(stderr, "malloc failed\n"); exit(1); } n->data = val; n->next = NULL; return n; }
📐 Memory Layout of a Node

On a 64-bit system: sizeof(Node) = 12 bytes (4 + 8) with alignment possibly making it 16 bytes.

🏠 Stack vs Heap
📦
Stack: Fixed-size, automatic. Local variables live here. Freed when function returns. Fast but limited.
Node n; // stack n.data = 5; n.next = NULL; // freed when out of scope!
🌐
Heap: Dynamic, manual. malloc() allocates, free() deallocates. Lives until explicitly freed.
Node* n = malloc(...); // heap n->data = 5; n->next = NULL; free(n); // YOU must free it

🧹 malloc and free
/* Allocating a node */ Node* p = (Node*) malloc(sizeof(Node)); /* Free a single node */ free(p); p = NULL; // prevent dangling pointer /* Free entire list — traverse and free each */ void freeList(Node* head) { Node* curr = head; while (curr != NULL) { Node* temp = curr->next; // save next BEFORE freeing free(curr); curr = temp; } }
💥
Memory leak! If you lose all references to a heap node without calling free(), that memory is leaked. Over time this can crash long-running programs.

➕ Insert at Position
void insertAt(Node** head, int val, int pos) { if (pos == 0) { insertBeg(head, val); return; } Node* n = createNode(val); Node* t = *head; for (int i = 0; i < pos-1 && t != NULL; i++) t = t->next; if (t == NULL) { free(n); return; } // out of range n->next = t->next; t->next = n; }
🗑️ Delete at Beginning
void deleteBeg(Node** head) { if (*head == NULL) return; // empty Node* temp = *head; // save ref *head = (*head)->next; // move head free(temp); // release memory }
🗑️ Delete at End
void deleteEnd(Node** head) { if (*head == NULL) return; if ((*head)->next == NULL) { free(*head); *head = NULL; return; } Node* t = *head; while (t->next->next != NULL) t = t->next; free(t->next); t->next = NULL; }
🗑️ Delete at Position
void deleteAt(Node** head, int pos) { if (*head == NULL) return; if (pos == 0) { deleteBeg(head); return; } Node* t = *head; for (int i = 0; i < pos-1 && t->next; i++) t = t->next; if (!t->next) return; // out of range Node* del = t->next; t->next = del->next; free(del); }
🚶 Traverse List
Speed:
void traverse(Node* head) { Node* curr = head; while (curr != NULL) { printf("%d → ", curr->data); curr = curr->next; } printf("NULL\n"); }
🔍 Search by Value
int search(Node* head, int key) { Node* curr = head; int idx = 0; while (curr != NULL) { if (curr->data == key) return idx; // found curr = curr->next; idx++; } return -1; // not found }

🔄 Reverse Linked List

Time: O(n) Space: O(1) Easy

Reverse a singly linked list in-place using three pointers: prev, curr, next. Classic interview question.

Press ▶ to start animation
Speed:
Node* reverse(Node* head) { Node* prev = NULL; Node* curr = head; Node* next = NULL; while (curr != NULL) { next = curr->next; /* save next */ curr->next = prev; /* reverse link */ prev = curr; /* move prev */ curr = next; /* move curr */ } return prev; /* new head */ }

🐢🐇 Find Middle (Slow/Fast Pointer)

Time: O(n) Space: O(1) Easy

Use two pointers: slow moves 1 step, fast moves 2 steps. When fast reaches the end, slow is at the middle. Also works for cycle detection.

Press ▶ to start animation
Speed:
Node* findMiddle(Node* head) { Node* slow = head; Node* fast = head; while (fast != NULL && fast->next != NULL) { slow = slow->next; /* +1 step */ fast = fast->next->next; /* +2 steps */ } return slow; /* middle node */ }

🔁 Detect Cycle — Floyd's Algorithm

Time: O(n) Space: O(1) Medium

Floyd's "tortoise and hare" algorithm. If a cycle exists, slow and fast pointers will eventually meet inside the cycle.

Press ▶ to animate cycle detection
int hasCycle(Node* head) { Node* slow = head; Node* fast = head; while (fast != NULL && fast->next != NULL) { slow = slow->next; fast = fast->next->next; if (slow == fast) return 1; /* cycle detected! */ } return 0; /* no cycle */ }

🔤 Palindrome Linked List

Time: O(n) Space: O(1) Medium

Find middle → reverse second half → compare from both ends. O(1) space solution using in-place reversal.

Press ▶ to start
int isPalindrome(Node* head) { if (!head || !head->next) return 1; /* Step 1: find middle */ Node *slow = head, *fast = head; while (fast->next && fast->next->next) { slow = slow->next; fast = fast->next->next; } /* Step 2: reverse second half */ Node* second = reverse(slow->next); /* Step 3: compare */ Node *p1 = head, *p2 = second; int result = 1; while (p2) { if (p1->data != p2->data) { result = 0; break; } p1 = p1->next; p2 = p2->next; } reverse(second); /* restore list */ return result; }

🎲 Odd-Even Linked List (by index)

Time: O(n) Space: O(1) Medium

Group nodes at odd indices first, then even-index nodes. Rearrange by pointer manipulation without creating new nodes.

Press ▶ to start
Node* oddEvenList(Node* head) { if (!head) return head; Node *odd = head; Node *even = head->next; Node *evenHead = even; while (even && even->next) { odd->next = even->next; /* link odd nodes */ odd = odd->next; even->next = odd->next; /* link even nodes */ even = even->next; } odd->next = evenHead; /* join lists */ return head; }
